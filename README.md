## 💡 一句话说清楚

你记下来的内容会**随着时间慢慢"坏掉"**：字会消失、数字会变错、句子会变得模糊不清。

想要让它们恢复原状？唯一的方法就是——**你自己回忆起来，填上去**。

它不追求帮你"存住"信息，而是**逼你"记住"信息**。

## 🤔 为什么需要这样一个工具

### 现状的问题

- 大部分人记笔记＝把信息从别处复制粘贴到一个地方
- 记完就再也不看——"存了就等于会了"是大脑最大的骗局
- 现有的学习工具（背单词 App、题库）和"你自己记的笔记"是分裂的
- 考试前翻笔记，发现跟看陌生人写的一样——因为从没主动回忆过

### FadeMemo 想做的事

- 把你的笔记变成一个"活的东西"——不看它就会生锈
- 每次修复都是一次主动回忆，这是公认最高效的学习方式
- 让"记"和"学"合二为一，不再分裂

## 🧠 核心理念：存储和记忆是矛盾的

传统笔记工具的底层逻辑是：只要存得够完整、够精确，就永远不需要去记它。

但现实是：

- 你记了密码管理器里所有密码——但你一个也背不出来
- 你存了整本教材的笔记——但考试时什么都想不起来
- 你保存了某次对话的完整聊天记录——但需要时还是得去搜

**存储越完美，大脑越不需要参与。大脑越不参与，记忆越脆弱。**

FadeMemo 反过来：笔记越完整，你越不需要去记它；笔记越残缺，你越需要去回忆它。前者让你省心但忘记，后者让你费力但记住。

> 🎯 工具的目标不是"帮你记住一切"，而是"让你真正记得住你在乎的东西"。

## 🔧 损坏引擎

### 艾宾浩斯遗忘曲线

损坏引擎直接套用艾宾浩斯遗忘曲线公式：

```
记忆保留率 = e^( - 距离上次回顾的时间 / 笔记的"记忆强度" )
```

它的行为很简单：

- **新笔记**：昨天还记得清清楚楚，今天不看就模糊了一小块
- **常回顾的老笔记**：一星期不看也只掉了一点点
- **答错一次**：笔记的记忆强度减半——损坏得越来越快

这准确地模拟了真实记忆的状态：**你越不去想的，丢得越快；你想错了的，会干扰正确的记忆。**

### 损坏等级（0–10 级）

损坏不是"有/无"的开关，而是渐变的过程：

| 等级 | 状态 | 说明 |
| :---: | :--- | :--- |
| 0 | 完好无损 | 内容完全正常，和刚写的时候一样 |
| 1 | 轻微磨损 | 有一两个数字偏了一点点（1789 → 1790） |
| 2 | 轻度损坏 | 关键数字偏了 3–5，或者有个别错别字 |
| 3 | 明显受损 | 核心概念被替换成近似的（巴士底狱 → 凡尔赛宫） |
| 4 | 比较严重 | 时间、地点、人名几处同时出错 |
| 5 | 显著损坏 | 核心信息全错，只剩几个无关紧要的词能看懂 |
| 6 | 重度损坏 | 只剩不到一半的内容是原样 |
| 7 | 非常严重 | 只剩半句话和一个标题 |
| 8 | 临界状态 | 只剩标题和标签 |
| 9 | 濒危 | 只显示"这是一条关于 XXX 的笔记" |
| 10 | 彻底遗忘 | "你已经忘了这条笔记的内容" |

### 等级的升降

**答对的时候：**

- 第一次就答对 → 等级 **降 2 级**
- 第二次答对 → 等级 **降 1 级**

**答错的时候：**

- 第一次答错 → 等级 **升 1 级**，并立刻展示正确答案
- 连续三次答错 → 等级 **升 3 级**，强制展示完整原文
- 答错后的"纠正对比"是强化记忆的最佳时机

**自然遗忘（你什么都不做的时候）：**

- 超过 24 小时 → 自动升 1 级
- 超过 48 小时 → 再升 2 级
- 超过 72 小时 → 再升 3 级

> ⚖️ 答对的奖励（降 2 级）大于答错的惩罚（升 1 级）。只要你经常回顾，笔记就能维持健康状态。

## 🎮 五种记忆挑战模式

同样的笔记，可以用不同的方式"损坏"，每种方式训练不同的记忆能力。

### 1️⃣ 错误检测

故意把原文中的事实改成错的，你来判断对错。

> 你记了"法国大革命发生在 **1789 年**"，系统显示为"法国大革命发生在 **1744 年**"。
> 你能看出这是错的吗？

错误的设计原则：**似是而非**。太假了没有学习效果，太真了是陷阱。

### 2️⃣ 信息填空

挖掉关键信息，你来填。工具优先挖掉：**时间、地点、人名、数字、关键概念**。

> 法国大革命发生于 **____** 年，**____** 月 **____** 日，**____** 被攻占。

- 填对 → 大幅加强记忆强度
- 填错 → 显示正确答案对比，纠正错误记忆
- 连续填错 3 次 → "别猜了，先看看再回来答"

### 3️⃣ 对比判断

同时展示两段内容，一段真一段假，你来选出真的。

> **卡片 A：** 法国大革命发生在 1789 年，巴士底狱被攻占
> **卡片 B：** 法国大革命发生在 1789 年，凡尔赛宫被攻占

两种信息都"看起来挺对"的——但其中一个细节是错的。适合容易混淆的知识点。

### 4️⃣ 时序重组

如果笔记包含多个按时间发生的事件，打乱顺序，你来重新排列。

> A. 拿破仑称帝 → B. 攻占巴士底狱 → C. 法兰西第一共和国成立
> **正确顺序：** B → C → A

适合历史事件链、操作步骤、流程说明——测试的是因果理解，而不是孤立记忆。

### 5️⃣ 缩写还原

给你一个缩略版本（骨架），你把它展开成完整表述。

> **缩写版：**"1789，法革爆发，巴士底狱被攻占"
> **完整版：**"1789 年 7 月 14 日，法国大革命爆发，巴黎人民攻占巴士底狱"

帮你把被动记忆变成主动输出。

## 🏗️ 技术构成

纯本地运行，不需要服务器、不需要注册账号、不需要联网。

| 层次 | 技术 | 职责 |
| :--- | :--- | :--- |
| **界面** | **Flutter** | 写笔记、看笔记损坏的样子、填空修复。跨平台（Windows/Mac/Linux/手机） |
| **大脑** | **Go** | 损坏引擎——计算艾宾浩斯遗忘曲线、挖空方案、评分。启动快、资源占用低 |
| **存储** | **Hive** | 轻量级本地数据库，嵌入在界面层。所有笔记存在你的电脑上 |

### 数据流

```
你写笔记 → Hive 存原始内容
        ↓
打开笔记 → Flutter 发给 Go 大脑
        ↓
Go 按艾宾浩斯曲线计算损坏方案
        ↓
Go 返回损坏方案 → Flutter 展示损坏效果
        ↓
你填空/判断 → 结果发给 Go 评分 → 更新记忆强度
```

> 💡 **核心内容永不丢失。** 损坏只是展示层面的"伪装"——Hive 数据库里存的是完好的原始内容，损坏只在你看的时候实时计算。关掉"损坏模式"，笔记完好如初。

## 🤖 为什么先不用 AI？

规则引擎在前，AI 在后。

| 问题 | 说明 |
| :--- | :--- |
| **不确定性** | 规则驱动下逻辑清晰、预期明确；AI 驱动下用户无法建立信任 |
| **AI 猜得不准** | AI 可能把"地球围绕太阳转"挖空了——规则引擎不会犯这种低级错误 |
| **离线能力** | 本地跑 AI 需下载 2–4GB 模型，笔记本风扇起飞；规则引擎毫秒级响应 |
| **复杂度陷阱** | 核心机制先做对、做稳，比堆新功能重要得多 |

> 🚩 **决定：先做规则引擎，跑稳了以后再考虑让 AI 辅助出题。规则兜底，AI 锦上添花。**

## ❓ 常见问题

### 1. "记了也白记"的恐惧

> 我认认真真写了几百条笔记，过段时间全坏了怎么办？

**核心内容永不丢失。** 损坏只是展示层面的"伪装"——Hive 数据库里存的是完好的原始内容，损坏只在你看的时候实时计算。关掉"损坏模式"，笔记完好如初。

### 2. 标记关键信息太麻烦

> 如果每次写笔记都要手动圈重点，大部分人坚持不了三天。

**编辑器自动检测并标记。** 你写"1789 年 7 月 14 日"，系统自动识别这是一个日期并标注为关键信息——你看一眼，觉得不对可以点掉，零额外操作。

### 3. 答错挫败感太强

> 如果答错惩罚太重，用户直接删掉这个工具。

**答对奖励（降 2 级）＞答错惩罚（升 1 级）。** 坚持回顾的人会发现笔记越来越"耐用"——花时间就有回报。

### 4. 适用范围窄

> 不是所有笔记都适合"被损坏"。密码、购物清单、收货地址——这些只需要存，不需要记。

**每条笔记可以有"类型"标签。** 学习型笔记启用损坏模式，工具型笔记永远完好。

### 5. 多设备同步

第一版只做本地单设备，不做同步。同步涉及复杂的冲突处理（A 设备损坏到 level 7，B 设备修复到 level 2，以谁为准？），等产品稳定后再考虑。

## 💬 最后说几句

这个项目最特别的地方是：它对人诚实。

市面上几乎所有笔记工具都在做同一件事——帮你把信息存得更好、更快、更全。但它们从来不告诉你一个真相：**存下来 ≠ 学会了**。

FadeMemo 不追求"帮你存"，它追求"让你记住"。
它不给你虚假的安全感。它会直白地告诉你：你不回顾，就会失去。

这听上去有点残忍，但这就是真实记忆的运作方式。

你很奇怪笔记为什么能记住你忘了什么？
它不能。它只是诚实地提醒了你。
