# FadeMemo 记忆遗忘模拟器

FadeMemo 是一款模拟人类记忆自然遗忘规律的跨端笔记 App。它不把“永久保存”作为核心卖点，而是让文字随着时间逐渐模糊、错乱、掉字，直至消失，用可视化的方式呈现记忆被时间侵蚀的过程。

项目采用 Flutter 一套代码覆盖 Windows、macOS、Android 与 iOS。核心功能优先离线可用，后端仅作为多端同步、云端备份、用户体系和笔记分享的进阶能力。

## 项目定位

传统笔记软件通常强调稳定存储、快速检索和长期归档。FadeMemo 选择另一条路径：文字会像记忆一样衰减，用户需要通过“回忆”让它重新清晰，也可以主动让它更快遗忘。

核心特色包括：

- 根据艾宾浩斯遗忘曲线模拟文字衰减
- 以单个字符为最小状态单元，而不是整段文本统一变化
- 支持文字模糊、错乱、掉字和消失等视觉退化效果
- 支持点击或 hover 字符来“回忆擦亮”
- 支持一键加速失忆、重置记忆和调节全局遗忘速度
- 使用低饱和、极简、安静的视觉风格营造文艺氛围

## 支持平台

FadeMemo 计划支持以下平台：

- Windows 桌面端
- macOS 桌面端
- Android 手机端
- iOS 手机端

所有平台使用 Flutter 统一实现 UI、动画和交互，尽量保持手机与桌面端体验一致。

## 技术栈

### 客户端

- UI 框架：Flutter
- UI 实现方式：纯 Flutter 自绘 UI
- 适配目标：手机触屏、桌面鼠标 hover、全窗口自适应
- 动画目标：状态变化平滑过渡，无闪烁、无突兀跳变

### 本地存储

| 用途 | 技术 | 说明 |
| --- | --- | --- |
| 主力动态数据存储 | Hive | 存储整篇笔记、逐字字符状态、时间戳、记忆强度等高频变化数据 |
| 结构化拓展存储 | SQLite / Drift | 后续用于笔记分类、归档、检索等关系型数据 |
| 轻量配置存储 | SharedPreferences | 存储全局遗忘速度、主题模式、动画开关等基础设置 |

### 可选后端

- 后端框架：Golang + Gin
- 后端定位：多端同步、云端备份、用户体系、笔记分享
- 实时同步：WebSocket
- 云端数据库：MongoDB Atlas

单机版本不依赖后端，断网时仍可正常使用全部核心功能。

## 核心数据结构

FadeMemo 的最小状态颗粒度是单个字符。每个字符都拥有独立的记忆强度、重要性和时间信息。

```json
{
  "char": "记",
  "strength": 1.0,
  "importance": 0.8,
  "createTime": 1720000000000,
  "lastRecall": 1720000000000
}
```

字段说明：

- `char`：字符本体
- `strength`：记忆强度，范围为 `0 ~ 1`，`1` 表示清晰，`0` 表示消失
- `importance`：字符重要性，范围为 `0 ~ 1`，数值越高遗忘越慢
- `createTime`：字符创建时间戳
- `lastRecall`：最后一次被回忆的时间戳

## 遗忘算法

FadeMemo 使用基于艾宾浩斯遗忘曲线的指数衰减模型，并根据字符重要性调整遗忘速度。

```text
strength(t) = importance × exp( -λ × (t - createdAt)^β )
```

参数含义：

- `strength(t)`：字符在时间 `t` 的记忆强度
- `importance`：字符重要性，重要字符衰减更慢
- `λ`：全局遗忘系数，由用户通过滑块调节
- `β`：衰减曲率，不重要字符的 `β` 更大，衰减更快
- `createdAt`：字符创建时间

当用户执行“回忆擦亮”操作时，字符的 `strength` 会提升，视觉状态随之恢复清晰；当用户执行“加速失忆”操作时，字符强度会整体降低，文字更快进入模糊、错乱或消失状态。

## 视觉退化机制

字符会根据 `strength` 自动进入四种视觉状态。

| 状态 | 强度范围 | 表现 |
| --- | --- | --- |
| 清晰态 | `0.7 ~ 1.0` | 正常透明度、无模糊、轻微呼吸动画 |
| 模糊态 | `0.4 ~ 0.7` | 透明度降低、高斯模糊、微小位置抖动 |
| 错乱态 | `0.1 ~ 0.4` | 字符随机形变、偏旁替换、RGB 色偏故障风效果 |
| 消失态 | `0 ~ 0.1` | 完全透明、脱离文本流、保留空白占位 |

所有视觉效果都应逐帧动态渲染，并通过平滑动画完成状态过渡。

## 核心交互

- 回忆擦亮：点击或 hover 某个字符，提升该字符的 `strength`，使其恢复清晰
- 加速失忆：一键降低全局字符强度，让内容快速进入模糊或错乱状态
- 重置记忆：将所有字符恢复到 `100%` 清晰状态
- 速度调节：通过全局滑块控制整体遗忘速率
- 明暗模式：在安静、低饱和的视觉系统中切换浅色与深色主题

## UI 与体验规范

FadeMemo 的界面应保持极简、安静、文艺和低饱和度，不使用冗余按钮和复杂弹窗。页面固定分为三个模块：

1. 顶部：文本输入编辑区，用于新增或修改笔记
2. 中部：核心遗忘展示区，逐字动态渲染所有视觉特效
3. 底部：控制面板，包含遗忘速度、回忆、失忆、重置和明暗模式切换

桌面端需要支持鼠标 hover 触发回忆效果，移动端需要适配触摸交互。不同屏幕尺寸下，布局应保持内容可读、控制清晰、动画连续。

## 开发阶段

### 阶段 1：纯单机完整版

第一阶段优先实现完整离线体验：

- Flutter 客户端完整 UI
- 文本逐字拆解与字符状态管理
- Hive 本地持久化
- 遗忘算法
- 四层视觉退化特效
- 回忆擦亮、加速失忆、重置记忆、速度调节
- SharedPreferences 配置存储
- 明暗主题切换
- Windows、macOS、Android、iOS 全平台适配与打包

### 阶段 2：进阶云端版

第二阶段在单机版稳定后再引入后端：

- Golang + Gin 后端服务
- MongoDB Atlas 云端数据库
- 多设备数据同步
- 云端备份
- WebSocket 实时状态同步
- 用户体系
- 笔记分享

## 关键约束

- 核心功能必须优先离线可用，网络只用于同步和备份
- 最小状态颗粒度必须是单字符，禁止按段落统一处理状态
- 客户端必须使用 Flutter 纯自绘 UI，不使用网页套壳方案
- 本地动态数据优先使用 Hive
- SQLite / Drift 只用于后续结构化能力拓展
- 后端仅使用 Golang + Gin，不使用 Java、PHP、Express 等其他框架
- 云端数据库使用 MongoDB Atlas，与本地 Hive 的文档型结构保持一致

## 项目愿景

FadeMemo 希望把笔记从“信息仓库”变成“记忆现场”。用户写下的文字不会被机械地永久保存，而会以可见的方式被时间改变。每一次回忆、遗忘和重置，都是用户与文本之间的一次重新连接。
